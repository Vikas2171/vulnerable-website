Cross-Site Request Forgery (CSRF) via GET

What is CSRF via GET?

A state-changing action (like a transfer) is performed via a GET request. An attacker can craft a malicious URL that, when visited by a logged-in user, will execute the action without their consent.

How to exploit:

  • Create a malicious link on an external website:
  • <img src="http://localhost:3000/csrf-get-action?amount=1000&recipient=attacker@evil.com">
